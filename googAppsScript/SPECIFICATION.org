* 1. Executive Summary

This document defines the functional specification for a Claude Code skill that
enables automated integration with Google Apps Script (GAS). The skill allows
Claude Code to write, upload, execute, test, and iteratively debug Apps Script
code directly from the terminal, creating a seamless development loop without
manual interaction with the Apps Script web editor.

The orchestration layer is implemented in Python and runs on Debian Linux. It
wraps the =clasp= CLI and =git= commands in Python subprocess calls, providing
structured error handling, log parsing, and state management that Claude Code
can invoke through shell commands.

The workflow is built on a Git-based branching strategy that ensures working
production code is never modified directly. All new features, fixes, and
experiments occur on isolated branches. Code only reaches the production
Apps Script deployment after passing tests and being merged through a
controlled process.

The intended audience for this document is an AI agent (Claude Code) that will
use it as a reference to implement, maintain, and extend the system.

* 2. Goals and Objectives

** 2.1 Primary Goals

The system must achieve the following outcomes:

- Eliminate manual copy-paste workflows between local development and the Apps Script editor.
- Enable Claude Code to autonomously push code, execute functions, capture results, and iterate based on feedback.
- Provide structured error reporting so Claude Code can diagnose and fix issues without human intervention.
- Support the full development lifecycle: scaffolding, writing, deploying, testing, and debugging.
- Protect production code through Git-based version control, branch isolation, and controlled merge workflows.
- Maintain a complete auditable history of all changes to the Apps Script codebase.
- Implement the orchestration layer in Python using only standard library modules and well-established packages available on Debian Linux.

** 2.2 Non-Goals

The following are explicitly out of scope for the initial version:

- GUI or web-based interfaces.
- Multi-user collaboration or shared project management.
- Deployment of Apps Script add-ons to the Google Workspace Marketplace.
- Management of Google Cloud Platform resources beyond what is required for Apps Script execution.
- GitHub Actions or CI/CD pipeline automation (the skill itself orchestrates the workflow).
- Replacing =clasp= with direct Apps Script API calls (clasp remains the transport layer).

** 3.2 Component Descriptions

*** 3.2.1 Skill Layer

The entry point registered with Claude Code. It defines the skill's name,
description, trigger conditions, and the instructions Claude Code follows
when the skill is activated. This is stored as a markdown file in the
=.claude/skills/= directory. The skill file instructs Claude Code to invoke
the Python CLI tool for all operations.

*** 3.2.2 Python CLI Entrypoint (=gas_skill.py=)

The main executable that Claude Code calls from the terminal. It uses
Python's =argparse= module to expose subcommands that map to workflow
phases. All output is structured as JSON to stdout so Claude Code can
parse results programmatically.

*** 3.2.3 Git Manager (=git_manager.py=)

A Python module that wraps Git and GitHub CLI commands via =subprocess=.
It manages branch creation, switching, committing, pushing, merging,
tagging, and release creation. It enforces the branching strategy defined
in Section 5 and automatically swaps =.clasp.json= when branches change.

*** 3.2.4 clasp Wrapper (=clasp_wrapper.py=)

A Python module that wraps all =clasp= CLI operations via =subprocess=.
It handles push, pull, run, logs, create, clone, deploy, and status
commands. It captures stdout, stderr, and exit codes and returns them
as structured Python objects.

*** 3.2.5 Log Parser (=log_parser.py=)

A Python module that parses the output of =clasp run= and =clasp logs=
into structured data. It extracts error types, stack traces, execution
time, return values, and quota warnings. It classifies errors using
the taxonomy defined in Section 7.8.

*** 3.2.6 Feedback Loop Engine (=runner.py=)

The core orchestration module that implements the iterative cycle of
push, run, evaluate, fix, and repeat. It calls into the other modules
and manages state transitions. It enforces retry limits and produces
structured reports of each iteration. All fix attempts are committed
to Git so the history is preserved.

*** 3.2.7 Configuration Manager (=config.py=)

Manages project configuration including paths, environment detection,
retry limits, and clasp config file management. Reads from a
=gas_skill.toml= configuration file in the project root.

#+BEGIN_SRC python
"""models.py — Structured data types for the orchestration layer."""
from __future__ import annotations

import json
from dataclasses import dataclass, field, asdict
from datetime import datetime
from enum import Enum
from typing import Any


class Severity(str, Enum):
    DEBUG = "DEBUG"
    INFO = "INFO"
    WARNING = "WARNING"
    ERROR = "ERROR"
    CRITICAL = "CRITICAL"


class TestStatus(str, Enum):
    PASS = "PASS"
    FAIL = "FAIL"
    ERROR = "ERROR"


class ErrorCategory(str, Enum):
    AUTH = "AUTH"
    REFERENCE = "REFERENCE"
    TYPE = "TYPE"
    QUOTA = "QUOTA"
    TIMEOUT = "TIMEOUT"
    NETWORK = "NETWORK"
    SYNTAX = "SYNTAX"
    MANIFEST = "MANIFEST"
    UNKNOWN = "UNKNOWN"


@dataclass
class CommandResult:
    """Result of a subprocess call to clasp or git."""
    command: str
    exit_code: int
    stdout: str
    stderr: str
    success: bool
    duration_sec: float

    def to_json(self) -> str:
        return json.dumps(asdict(self), indent=2)


@dataclass
class LogEntry:
    """A single parsed log entry from clasp logs."""
    timestamp: str
    severity: Severity
    message: str
    function_name: str = ""
    stack_trace: str = ""


@dataclass
class TestResult:
    """Result of a single test function."""
    status: TestStatus
    function: str
    message: str
    timestamp: str
    stack: str = ""


@dataclass
class TestReport:
    """Aggregate result from runAllTests."""
    total: int
    passed: int
    failed: int
    errors: int
    success: bool
    details: list[TestResult] = field(default_factory=list)

    def to_json(self) -> str:
        return json.dumps(asdict(self), indent=2)


@dataclass
class IterationRecord:
    """Record of one push-run-fix iteration in the feedback loop."""
    iteration: int
    push_result: CommandResult | None = None
    test_report: TestReport | None = None
    error_category: ErrorCategory | None = None
    fix_description: str = ""
    committed: bool = False
    commit_hash: str = ""


@dataclass
class LoopReport:
    """Final report from the feedback loop engine."""
    success: bool
    total_iterations: int
    iterations: list[IterationRecord] = field(default_factory=list)
    final_commit: str = ""
    branch: str = ""

    def to_json(self) -> str:
        return json.dumps(asdict(self), indent=2, default=str)

A Google Cloud project must exist and be linked to the Apps Script project.

*** 4.4.2 Apps Script API

The Apps Script API must be enabled in the Google Cloud Console at:
=APIs & Services > Library > Apps Script API=.

*** 4.4.3 OAuth Consent Screen

An OAuth consent screen must be configured. For personal/development use,
the app can remain in "Testing" mode with the developer's account added

*** 4.4.4 OAuth Credentials

OAuth 2.0 Client ID credentials (type: Desktop App) must be created and
downloaded. The credential file is used by =clasp login= to authenticate.

** 4.5 Authentication Flow

#+BEGIN_SRC

Claude Code must verify that =~/.clasprc.json= exists and contains a valid
(non-expired) token before attempting any operations. If the token is

as browser-based OAuth cannot be completed from within Claude Code.

* 5. Version Control Strategy

** 5.1 Branching Model

The project uses a simplified Git Flow branching model with three branch
tiers. The core principle is that production code on =main= is never
edited directly. All work happens on isolated branches.

*** 5.1.1 Branch Tiers

| Branch           | Purpose                    | Pushes to GAS? | Protected? |
|------------------+----------------------------+-----------------+------------|
| =main=           | Production-ready code      | YES (deploy)    | YES        |
| =develop=        | Integration branch         | YES (testing)   | NO         |
| =feature/*=      | Individual features/fixes  | YES (dev test)  | NO         |

*** 5.1.2 Branch Naming Convention

| Branch Type | Pattern                    | Example                          |
|-------------+----------------------------+----------------------------------|
| Production  | =main=                     | =main=                           |
| Development | =develop=                  | =develop=                        |
| Feature     | =feature/<description>=    | =feature/add-email-report=       |
| Bugfix      | =bugfix/<description>=     | =bugfix/fix-date-parsing=        |
| Hotfix      | =hotfix/<description>=     | =hotfix/critical-quota-error=    |
| Experiment  | =experiment/<description>= | =experiment/new-api-approach=    |

** 5.2 Apps Script Project Mapping

*** 5.2.1 Multi-Project Strategy

Use separate Apps Script projects for production and development:

| Git Branch       | Apps Script Project    | Config File               |
|------------------+------------------------+---------------------------|
| =main=           | Production project     | =.clasp.prod.json=        |
| =develop=        | Development project    | =.clasp.dev.json=         |
| =feature/*=      | Development project    | =.clasp.dev.json=         |

The =.clasp.json= file is listed in =.gitignore= so each branch does not
overwrite the other's project binding. The Python orchestration layer
copies the correct config file to =.clasp.json= when switching branches.

*** 5.2.2 Branch Switching Protocol

Handled by =git_manager.py=:

#+BEGIN_SRC python
def switch_branch(self, branch: str) -> CommandResult:
    """Switch Git branch and update .clasp.json for the correct environment."""
    self._require_clean_tree()

    if branch == self.config.main_branch:
        config_source = self.project_root / self.config.prod_config
    else:
        config_source = self.project_root / self.config.dev_config

    target = self.project_root / ".clasp.json"

    if not config_source.exists():
        raise ConfigError(f"Missing clasp config: {config_source}")

    result = self._run_git(["checkout", branch])

    if result.success:
        shutil.copy2(config_source, target)

    return result
#+END_SRC

| Type       | When to Use                                        |
|------------+----------------------------------------------------|
| =feat=     | New feature or functionality                       |

*** 5.3.3 Commit Granularity

Claude Code must commit at these checkpoints: after initial code is
written (before first push), after each successful fix iteration in
the feedback loop, after tests pass, and before any merge operation.
Each commit must be atomic and represent a single logical change.

def cmd_init(args) -> dict:
    """
    Full project scaffold.

    Steps:
    1.  Validate prerequisites (run preflight)
    2.  Create production Apps Script project:
        clasp create --type {args.type} --title "{args.title}"
    3.  Save .clasp.json as .clasp.prod.json
    4.  Create development Apps Script project:
        clasp create --type {args.type} --title "{args.title} [DEV]"
    5.  Save .clasp.json as .clasp.dev.json
    6.  Set active config to dev:
        shutil.copy2(".clasp.dev.json", ".clasp.json")
    7.  Create directory structure:
        src/, tests/, gas_skill/, docs/
    8.  Generate initial files:
        - .gitignore (see Appendix)
        - .claspignore (see Appendix)
        - appsscript.json (V8 runtime)
        - gas_skill.toml
        - CHANGELOG.md
        - README.md
        - src/Code.js (stub)
        - src/Tests.js (test harness with runAllTests)

def _validate_pre_push(self) -> list[dict]:
    """
    Returns a list of validation results.
    Each result: {"check": str, "passed": bool, "detail": str}
    Raises PrePushValidationError if any critical check fails.
    """
    checks = []

    # 1. Branch check
    branch = self.git.current_branch()
    is_main = branch == self.config.main_branch
    checks.append({
        "check": "not_on_main",
        "passed": not is_main,
        "detail": f"Current branch: {branch}"
    })

    # 2. clasp config exists
    clasp_json = self.project_root / ".clasp.json"
    checks.append({
        "check": "clasp_config_exists",
        "passed": clasp_json.exists(),
        "detail": str(clasp_json)
    })

    # 3. Script ID matches environment
    if clasp_json.exists():
        active_id = self._read_script_id(clasp_json)
        expected_config = (
            self.config.prod_config if is_main else self.config.dev_config
        )
        expected_id = self._read_script_id(
            self.project_root / expected_config
        )
        checks.append({
            "check": "script_id_matches_env",
            "passed": active_id == expected_id,
            "detail": f"active={active_id[:12]}... expected={expected_id[:12]}..."
        })

    # 4. Clean working tree
    clean = self.git.is_clean()
    checks.append({
        "check": "clean_working_tree",
        "passed": clean,
        "detail": "All changes committed" if clean else "Uncommitted changes"
    })

    # 5. Valid manifest
    manifest = self.project_root / "appsscript.json"
    manifest_valid = self.manifest.validate(manifest)
    checks.append({
        "check": "valid_manifest",
        "passed": manifest_valid.ok,
        "detail": manifest_valid.message
    })

    return checks

def cmd_test_loop(args) -> LoopReport:
    """
    Feedback loop engine.

    Configuration (from gas_skill.toml or CLI args):
        max_retries:          5
        retry_delay_sec:      2
        log_wait_sec:         3
        test_runner_function: "runAllTests"

    Algorithm:

    iterations = []
    for attempt in range(1, max_retries + 1):
        record = IterationRecord(iteration=attempt)

        # --- Auto-commit if dirty ---
        if not git.is_clean():
            git.add_all()
            git.commit(f"fix(iter-{attempt}): pre-push auto-commit")
            record.committed = True
            record.commit_hash = git.head_hash()

        # --- Push ---
        push_result = clasp.push()
        record.push_result = push_result

        if not push_result.success:
            record.error_category = log_parser.classify_error(push_result.stderr)
            iterations.append(record)

            # Return structured error for Claude Code to fix
            return LoopReport(
                success=False,
                total_iterations=attempt,
                iterations=iterations,
                branch=git.current_branch(),
                final_commit=git.head_hash(),
                action_needed="FIX_PUSH_ERROR",
                error_detail=push_result.stderr
            )

        # --- Wait for propagation ---
        time.sleep(log_wait_sec)

        # --- Run tests ---
        run_result = clasp.run(test_runner_function)

        if run_result.success:
            test_report = log_parser.parse_test_output(run_result.stdout)
            record.test_report = test_report

            if test_report.success:
                # All tests passed
                git.add_all()
                git.commit(f"test(pass): all {test_report.total} tests pass")
                record.committed = True
                record.commit_hash = git.head_hash()
                git.push()

                iterations.append(record)
                return LoopReport(
                    success=True,
                    total_iterations=attempt,
                    iterations=iterations,
                    branch=git.current_branch(),
                    final_commit=git.head_hash()
                )
            else:
                # Tests failed — return details for Claude Code to fix
                record.error_category = ErrorCategory.TEST_FAILURE
                iterations.append(record)

                return LoopReport(
                    success=False,
                    total_iterations=attempt,
                    iterations=iterations,
                    branch=git.current_branch(),
                    final_commit=git.head_hash(),
                    action_needed="FIX_TEST_FAILURE",
                    error_detail=test_report.to_json()
                )
        else:
            # Run itself failed (not a test failure, an execution error)
            record.error_category = log_parser.classify_error(run_result.stderr)
            iterations.append(record)

            return LoopReport(
                success=False,
                total_iterations=attempt,
                iterations=iterations,
                branch=git.current_branch(),
                final_commit=git.head_hash(),
                action_needed="FIX_RUNTIME_ERROR",
                error_detail=run_result.stderr
            )

    # Should not reach here, but safety net
    return LoopReport(
        success=False,
        total_iterations=max_retries,
        iterations=iterations,
        branch=git.current_branch(),
        final_commit=git.head_hash(),
        action_needed="MAX_RETRIES_EXCEEDED"
    )
#+END_SRC

*** Feedback Loop Interaction Model

The test loop is designed to return control to Claude Code after each
failure so that Claude Code can apply intelligent fixes. The typical
interaction pattern is:

#+BEGIN_SRC
Claude Code                           gas-skill CLI
    │                                      │
    │  1. Write/modify source files        │
    │  2. gas-skill commit ...             │
    │──────────────────────────────────────▶│
    │                                      │ commit + push to GitHub
    │◀─────── JSON result ─────────────────│
    │                                      │
    │  3. gas-skill test-loop              │
    │──────────────────────────────────────▶│
    │                                      │ push to GAS
    │                                      │ run tests
    │◀─── JSON: FAIL + error details ──────│
    │                                      │
    │  4. Analyze error, modify code       │
    │  5. gas-skill commit ...             │
    │──────────────────────────────────────▶│
    │◀─────── JSON result ─────────────────│
    │                                      │
    │  6. gas-skill test-loop              │
    │──────────────────────────────────────▶│
    │                                      │ push to GAS
    │                                      │ run tests
    │◀─── JSON: PASS ─────────────────────│
    │                                      │
    │  7. gas-skill merge ...              │
    │──────────────────────────────────────▶│
    │                                      │
#+END_SRC

The key design decision is that Claude Code retains agency over the fix.
The =test-loop= command does not attempt to fix code itself. It pushes,
runs, and reports. Claude Code reads the structured error report, decides
on a fix, edits the files, commits, and runs the loop again. This keeps
Claude Code as the intelligent actor and the CLI as the reliable executor.

- Every code change (including fix attempts) is committed before the next push.
- The Git log on a feature branch shows the complete evolution of the code through the test loop.
- Maximum retry limit is enforced to prevent infinite loops.
- The test loop returns control to Claude Code after each failure rather than attempting automated fixes.

** 7.8 FR-008: Error Classification

*** Description
All errors from clasp operations must be classified into categories so
Claude Code can apply appropriate fix strategies.

*** Error Taxonomy

| Category      | Pattern / Indicator                           | Typical Fix                              |
|---------------+-----------------------------------------------+------------------------------------------|
| =AUTH=        | "not logged in", 401, token expired           | Re-authenticate (user action)            |
| =REFERENCE=   | "ReferenceError", "is not defined"            | Fix variable/function name               |
| =TYPE=        | "TypeError", "is not a function"              | Fix type mismatch or method call         |
| =SYNTAX=      | "SyntaxError", "Unexpected token"             | Fix JavaScript syntax                    |
| =MANIFEST=    | "Invalid manifest", malformed appsscript.json | Fix manifest JSON                        |
| =QUOTA=       | "quota", "rate limit", "RESOURCE_EXHAUSTED"   | Wait and retry, or optimize              |
| =TIMEOUT=     | "exceeded maximum execution time"             | Optimize code or break into batches      |
| =NETWORK=     | "ECONNREFUSED", "ETIMEDOUT", network errors   | Retry after delay                        |
| =PERMISSION=  | "permission denied", 403, insufficient scope  | Update OAuth scopes in manifest          |
| =NOT_FOUND=   | "script not found", 404                       | Check scriptId in .clasp.json            |
| =UNKNOWN=     | Anything not matching above patterns          | Return full error for manual analysis    |

/**
 * Tests.js — Lightweight test harness for Apps Script.
 *
 * Convention:
 *   - All test functions start with "test_"
 *   - Each test function returns {passed: bool, message: string}
 *   - runAllTests() discovers and runs all test_ functions
 *   - Results are returned as JSON for parsing by the Python orchestrator
 */

/**
 * Run all test functions and return structured results.
 * Called by: clasp run runAllTests
 * @returns {string} JSON string with test results
 */
function runAllTests() {
  var results = [];
  var totalPassed = 0;
  var totalFailed = 0;
  var totalErrors = 0;

  // Discover all test functions in the global scope
  var globalThis_ = this;
  var testFunctions = Object.keys(globalThis_).filter(function(name) {
    return name.indexOf("test_") === 0 && typeof globalThis_[name] === "function";
  });

  testFunctions.forEach(function(funcName) {
    var result = {
      function: funcName,
      status: "ERROR",
      message: "",
      timestamp: new Date().toISOString(),
      stack: ""
    };

    try {
      var outcome = globalThis_[funcName]();
      if (outcome && outcome.passed === true) {
        result.status = "PASS";
        result.message = outcome.message || "OK";
        totalPassed++;
      } else {
        result.status = "FAIL";
        result.message = (outcome && outcome.message) ? outcome.message : "Test returned falsy or missing .passed";
        totalFailed++;
      }
    } catch (e) {
      result.status = "ERROR";
      result.message = e.message || String(e);
      result.stack = e.stack || "";
      totalErrors++;
    }

    results.push(result);
  });

  var report = {
    total: results.length,
    passed: totalPassed,
    failed: totalFailed,
    errors: totalErrors,
    success: (totalFailed === 0 && totalErrors === 0 && results.length > 0),
    details: results
  };

  return JSON.stringify(report);
}

/**
 * Assert that two values are strictly equal.
 * @param {*} actual
 * @param {*} expected
 * @param {string} [label]
 * @returns {{passed: boolean, message: string}}
 */
function assertEqual(actual, expected, label) {
  var prefix = label ? label + ": " : "";
  if (actual === expected) {
    return { passed: true, message: prefix + "OK" };
  }
  return {
    passed: false,
    message: prefix + "Expected " + JSON.stringify(expected) +
             " but got " + JSON.stringify(actual)

 * Assert that a value is truthy.
 * @param {*} value

 * @returns {{passed: boolean, message: string}}

  if (value) {
    return { passed: true, message: prefix + "OK" };
  }
  return { passed: false, message: prefix + "Expected truthy, got " + JSON.stringify(value) };

 * Assert that a value is falsy.
 * @param {*} value
 * @param {string} [label]
 * @returns {{passed: boolean, message: string}}

  var prefix = label ? label + ": " : "";
  if (!value) {
    return { passed: true, message: prefix + "OK" };
  }
  return { passed: false, message: prefix + "Expected falsy, got " + JSON.stringify(value) };
}

/**
 * Assert that a function throws an error.
 * @param {Function} fn
 * @param {string} [expectedMessage] - Substring to match in error message
 * @param {string} [label]
 * @returns {{passed: boolean, message: string}}
 */
function assertThrows(fn, expectedMessage, label) {
  var prefix = label ? label + ": " : "";
  try {
    fn();
    return { passed: false, message: prefix + "Expected function to throw, but it did not" };
  } catch (e) {
    if (expectedMessage && e.message.indexOf(expectedMessage) === -1) {
      return {
        passed: false,
        message: prefix + "Threw, but message '" + e.message +
                 "' did not contain '" + expectedMessage + "'"
      };
    }
    return { passed: true, message: prefix + "Threw as expected: " + e.message };
  }
}

/**

 * @param {*} expected

function assertDeepEqual(actual, expected, label) {

  var actualStr = JSON.stringify(actual);
  var expectedStr = JSON.stringify(expected);

  }

    passed: false,
    message: prefix + "Deep equality failed.\n  Expected: " + expectedStr +

  };
}

def parse_test_output(clasp_stdout: str) -> TestReport:
    """
    Parse the JSON output from runAllTests into a TestReport.

    clasp run returns output in the format:
    > runAllTests
    {JSON string}

    We need to extract the JSON portion and parse it.
    """
    # clasp run prefixes output with "> functionName\n"
    lines = clasp_stdout.strip().splitlines()

    json_str = ""
    for i, line in enumerate(lines):
        stripped = line.strip()
        if stripped.startswith("{"):
            json_str = "\n".join(lines[i:])
            break

    if not json_str:
        return TestReport(
            total=0, passed=0, failed=0, errors=1,
            success=False,
            details=[TestResult(
                status=TestStatus.ERROR,
                function="runAllTests",
                message=f"Could not parse test output: {clasp_stdout[:200]}",
                timestamp="",
            )]
        )

    try:
        data = json.loads(json_str)
    except json.JSONDecodeError as e:
        return TestReport(
            total=0, passed=0, failed=0, errors=1,
            success=False,
            details=[TestResult(
                status=TestStatus.ERROR,
                function="runAllTests",
                message=f"Invalid JSON in test output: {e}",
                timestamp="",
            )]
        )

    details = [
        TestResult(
            status=TestStatus(d.get("status", "ERROR")),
            function=d.get("function", "unknown"),
            message=d.get("message", ""),
            timestamp=d.get("timestamp", ""),
            stack=d.get("stack", ""),
        )
        for d in data.get("details", [])
    ]

    return TestReport(
        total=data.get("total", 0),
        passed=data.get("passed", 0),
        failed=data.get("failed", 0),
        errors=data.get("errors", 0),
        success=data.get("success", False),

"""_subprocess.py — Shared subprocess execution utilities."""
import subprocess
import time
from pathlib import Path
from .models import CommandResult


def run_command(
    args: list[str],
    cwd: Path | None = None,
    timeout: int = 120,
    env: dict | None = None,
) -> CommandResult:
    """
    Execute a command and return a structured CommandResult.

    Args:
        args:    Command and arguments as a list.
        cwd:     Working directory for the command.
        timeout: Maximum seconds to wait before killing the process.
        env:     Optional environment variables (merged with os.environ).

    Returns:
        CommandResult with stdout, stderr, exit_code, and timing.
    """
    command_str = " ".join(args)
    start = time.monotonic()

    try:
        proc = subprocess.run(
            args,
            capture_output=True,
            text=True,
            cwd=cwd,
            timeout=timeout,
            env=env,
        )
        duration = time.monotonic() - start

        return CommandResult(
            command=command_str,
            exit_code=proc.returncode,
            stdout=proc.stdout.strip(),
            stderr=proc.stderr.strip(),
            success=(proc.returncode == 0),
            duration_sec=round(duration, 2),
        )

    except subprocess.TimeoutExpired:
        duration = time.monotonic() - start
        return CommandResult(
            command=command_str,
            exit_code=-1,
            stdout="",
            stderr=f"Command timed out after {timeout} seconds",
            success=False,
            duration_sec=round(duration, 2),
        )

    except FileNotFoundError:
        duration = time.monotonic() - start
        return CommandResult(
            command=command_str,
            exit_code=-1,
            stdout="",
            stderr=f"Command not found: {args[0]}",
            success=False,
            duration_sec=round(duration, 2),
        )

"""git_manager.py — Git and GitHub CLI operations with branch safety."""
import shutil
from pathlib import Path

from .models import CommandResult
from .config import ProjectConfig
from .exceptions import BranchProtectionError, DirtyTreeError, ConfigError


class GitManager:
    """Manages Git operations with branch strategy enforcement."""

    def __init__(self, config: ProjectConfig):
        self.config = config
        self.project_root = config.project_root

    # ── Query methods ──────────────────────────────────────────

    def current_branch(self) -> str:
        result = self._git(["branch", "--show-current"])
        return result.stdout.strip()

    def head_hash(self, short: bool = True) -> str:
        args = ["rev-parse"]
        if short:
            args.append("--short")
        args.append("HEAD")
        result = self._git(args)
        return result.stdout.strip()

    def is_clean(self) -> bool:
        result = self._git(["status", "--porcelain"])
        return result.stdout.strip() == ""

    def list_branches(self, remote: bool = False) -> list[str]:
        args = ["branch"]
        if remote:
            args.append("-a")
        result = self._git(args)
        return [
            line.strip().lstrip("* ")
            for line in result.stdout.splitlines()
            if line.strip()
        ]

    def latest_tag(self) -> str | None:
        result = self._git(["describe", "--tags", "--abbrev=0"])
        if result.success:
            return result.stdout.strip()
        return None

    def remote_sync_status(self) -> dict:
        self._git(["fetch", "origin"])
        branch = self.current_branch()
        ahead = self._git(
            ["rev-list", "--count", f"origin/{branch}..{branch}"]
        )
        behind = self._git(
            ["rev-list", "--count", f"{branch}..origin/{branch}"]
        )
        return {
            "ahead": int(ahead.stdout.strip()) if ahead.success else 0,
            "behind": int(behind.stdout.strip()) if behind.success else 0,
        }

    # ── Safety checks ─────────────────────────────────────────

    def _require_clean_tree(self) -> None:
        if not self.is_clean():
            raise DirtyTreeError("Working tree has uncommitted changes")

    def _require_not_main(self) -> None:
        if self.current_branch() == self.config.main_branch:
            raise BranchProtectionError(
                f"Direct commits to '{self.config.main_branch}' are not allowed"
            )

    def _require_branch_exists(self, branch: str) -> None:
        result = self._git(["rev-parse", "--verify", branch])
        if not result.success:
            raise ConfigError(f"Branch '{branch}' does not exist")

    # ── Mutation methods ───────────────────────────────────────

    def checkout(self, branch: str) -> CommandResult:
        self._require_clean_tree()
        result = self._git(["checkout", branch])
        if result.success:
            self._swap_clasp_config(branch)
        return result

    def create_branch(self, name: str, from_branch: str | None = None) -> CommandResult:
        self._require_clean_tree()
        if from_branch:
            self.checkout(from_branch)
            self._git(["pull", "origin", from_branch])
        result = self._git(["checkout", "-b", name])
        if result.success:
            self._swap_clasp_config(name)
            self._git(["push", "-u", "origin", name])
        return result

    def add_all(self) -> CommandResult:
        return self._git(["add", "-A"])

    def commit(self, message: str) -> CommandResult:
        self._require_not_main()
        return self._git(["commit", "-m", message])

    def push(self, branch: str | None = None) -> CommandResult:
        branch = branch or self.current_branch()
        return self._git(["push", "origin", branch])

    def pull(self, branch: str | None = None) -> CommandResult:
        branch = branch or self.current_branch()
        return self._git(["pull", "origin", branch])

    def merge(self, source: str, target: str, message: str) -> CommandResult:
        self.checkout(target)
        self.pull(target)
        return self._git(["merge", "--no-ff", source, "-m", message])

    def tag(self, version: str, message: str) -> CommandResult:
        return self._git(["tag", "-a", version, "-m", message])

    def push_tag(self, version: str) -> CommandResult:
        return self._git(["push", "origin", version])

    def delete_branch(self, branch: str, remote: bool = True) -> CommandResult:
        result = self._git(["branch", "-d", branch])
        if result.success and remote:
            self._git(["push", "origin", "--delete", branch])
        return result

    def abort_merge(self) -> CommandResult:
        return self._git(["merge", "--abort"])

    def reset_hard(self, ref: str = "HEAD") -> CommandResult:
        return self._git(["reset", "--hard", ref])

    # ── GitHub CLI methods ─────────────────────────────────────

    def gh_auth_status(self) -> CommandResult:
        return run_command(["gh", "auth", "status"], cwd=self.project_root)

    def gh_create_release(
        self, version: str, title: str, notes_file: str
    ) -> CommandResult:
        return run_command(
            [
                "gh", "release", "create", version,
                "--title", title,
                "--notes-file", notes_file,
            ],
            cwd=self.project_root,
        )

    # ── Internal helpers ───────────────────────────────────────

    def _git(self, args: list[str]) -> CommandResult:
        return run_command(["git"] + args, cwd=self.project_root)

    def _swap_clasp_config(self, branch: str) -> None:
        if branch == self.config.main_branch:
            source = self.project_root / self.config.prod_config
        else:
            source = self.project_root / self.config.dev_config

        target = self.project_root / ".clasp.json"

        if source.exists():
            shutil.copy2(source, target)
        else:
            raise ConfigError(f"Missing clasp config: {source}")

"""manifest.py — Manage appsscript.json manifest file."""
from __future__ import annotations

import json
from dataclasses import dataclass
from pathlib import Path


@dataclass
class ManifestValidation:
    """Result of manifest validation."""
    ok: bool
    message: str
    warnings: list[str]


# Required fields in appsscript.json
REQUIRED_FIELDS = {"timeZone", "exceptionLogging"}

# Default manifest for new projects
DEFAULT_MANIFEST = {
    "timeZone": "America/New_York",
    "dependencies": {},
    "exceptionLogging": "STACKDRIVER",
    "runtimeVersion": "V8",
    "oauthScopes": []
}


def validate(manifest_path: Path) -> ManifestValidation:
    """
    Validate an appsscript.json file.

    Checks:
    1. File exists
    2. File is valid JSON
    3. Required fields are present
    4. runtimeVersion is V8
    5. exceptionLogging is STACKDRIVER (recommended for log capture)
    """
    warnings: list[str] = []

    if not manifest_path.exists():
        return ManifestValidation(
            ok=False,
            message=f"Manifest not found: {manifest_path}",
            warnings=warnings,
        )

    try:
        with open(manifest_path) as f:
            data = json.load(f)
    except json.JSONDecodeError as e:
        return ManifestValidation(
            ok=False,
            message=f"Invalid JSON in manifest: {e}",
            warnings=warnings,
        )

    # Check required fields
    missing = REQUIRED_FIELDS - set(data.keys())
    if missing:
        return ManifestValidation(
            ok=False,
            message=f"Missing required fields: {', '.join(sorted(missing))}",
            warnings=warnings,
        )

    # Check runtime version
    if data.get("runtimeVersion") != "V8":
        warnings.append(
            f"runtimeVersion is '{data.get('runtimeVersion')}', "
            "expected 'V8' for modern JavaScript support"
        )

    # Check exception logging
    if data.get("exceptionLogging") != "STACKDRIVER":
        warnings.append(
            "exceptionLogging should be 'STACKDRIVER' for clasp logs to work"
        )

    return ManifestValidation(
        ok=True,
        message="Manifest is valid",
        warnings=warnings,
    )


def create_default(manifest_path: Path, timezone: str = "America/New_York") -> None:
    """Create a default appsscript.json."""
    manifest = DEFAULT_MANIFEST.copy()
    manifest["timeZone"] = timezone
    with open(manifest_path, "w") as f:
        json.dump(manifest, f, indent=2)
        f.write("\n")


def add_scope(manifest_path: Path, scope: str) -> None:
    """Add an OAuth scope to the manifest if not already present."""
    with open(manifest_path) as f:
        data = json.load(f)

    scopes = data.get("oauthScopes", [])
    if scope not in scopes:
        scopes.append(scope)
        data["oauthScopes"] = scopes
        with open(manifest_path, "w") as f:
            json.dump(data, f, indent=2)
            f.write("\n")

    "feat": "Added",
    "fix": "Fixed",
    "refactor": "Changed",
    "perf": "Changed",
    "docs": "Changed",
    "chore": "Changed",
    "style": "Changed",

def add_entry(
    changelog_path: Path,
    commit_type: str,
    description: str,
) -> None:
    """
    Add an entry to the [Unreleased] section of CHANGELOG.md.

    The entry is placed under the appropriate subsection header
    (Added, Fixed, Changed, etc.) based on the commit type.
    """
    if not changelog_path.exists():
        create_initial(changelog_path)

    content = changelog_path.read_text()
    section = TYPE_TO_SECTION.get(commit_type, "Changed")

    # Find [Unreleased] section
    unreleased_pattern = re.compile(r"(## \[Unreleased\]\n)", re.MULTILINE)
    match = unreleased_pattern.search(content)

    if not match:
        # No [Unreleased] section found; insert one after the header
        header_end = content.find("\n\n") + 2
        content = (
            content[:header_end]
            + "## [Unreleased]\n\n"
            + content[header_end:]
        )
        match = unreleased_pattern.search(content)

    # Find or create the subsection (e.g., ### Added) within [Unreleased]
    unreleased_start = match.end()

    # Find the end of the [Unreleased] section (next ## heading or EOF)
    next_version = re.search(r"\n## \[", content[unreleased_start:])
    if next_version:
        unreleased_end = unreleased_start + next_version.start()
    else:
        unreleased_end = len(content)

    unreleased_block = content[unreleased_start:unreleased_end]

    subsection_pattern = re.compile(rf"(### {section}\n)", re.MULTILINE)
    sub_match = subsection_pattern.search(unreleased_block)

    entry_line = f"- {description}\n"

    if sub_match:
        # Insert after the subsection header
        insert_pos = unreleased_start + sub_match.end()
        content = content[:insert_pos] + entry_line + content[insert_pos:]
    else:
        # Create the subsection at the end of [Unreleased]
        new_subsection = f"\n### {section}\n{entry_line}"
        content = (
            content[:unreleased_end]
            + new_subsection
            + content[unreleased_end:]
        )

    changelog_path.write_text(content)


def finalize_release(
    changelog_path: Path,
    version: str,
    release_date: date | None = None,
) -> None:
    """
    Move all [Unreleased] entries under a new version heading.

    Before:
        ## [Unreleased]
        ### Added
        - New feature

    After:
        ## [Unreleased]

        ## [1.2.0] - 2026-03-22
        ### Added
        - New feature
    """
    if release_date is None:
        release_date = date.today()

    content = changelog_path.read_text()

    # Extract [Unreleased] content
    unreleased_header = re.compile(r"## \[Unreleased\]\n", re.MULTILINE)
    match = unreleased_header.search(content)

    if not match:
        return

    unreleased_start = match.end()

    # Find next version heading
    next_version = re.search(r"\n## \[", content[unreleased_start:])
    if next_version:
        unreleased_end = unreleased_start + next_version.start()
    else:
        unreleased_end = len(content)

    unreleased_content = content[unreleased_start:unreleased_end].strip()

    if not unreleased_content:
        return  # Nothing to release

    # Build new version section
    version_heading = f"## [{version}] - {release_date.isoformat()}"
    new_section = f"\n\n{version_heading}\n{unreleased_content}\n"

    # Replace: clear [Unreleased] and insert new version section
    new_content = (
        content[:unreleased_start]
        + "\n"
        + new_section
        + content[unreleased_end:]
    )

    changelog_path.write_text(new_content)


def get_release_notes(changelog_path: Path, version: str) -> str:
    """
    Extract the changelog entries for a specific version.

    Returns the content between the version heading and the next
    version heading (or EOF), suitable for GitHub release notes.
    """
    content = changelog_path.read_text()

    version_pattern = re.compile(
        rf"## \[{re.escape(version)}\].*?\n(.*?)(?=\n## \[|\Z)",
        re.DOTALL,
    )
    match = version_pattern.search(content)

    if match:
        return match.group(1).strip()
    return f"Release {version}"

def cmd_merge(args: argparse.Namespace, config: ProjectConfig) -> None:
    """Merge branches."""
    git = GitManager(config)
    clasp = ClaspWrapper(config)

    merge_msg = f"merge({args.to}): integrate {args.from_branch}"

    try:
        result = git.merge(args.from_branch, args.to, merge_msg)
    except Exception as e:
        _error("Merge failed", str(e))
        return

    if not result.success:
        # Check for conflicts
        if "CONFLICT" in result.stdout or "CONFLICT" in result.stderr:
            conflict_result = git._git(["diff", "--name-only", "--diff-filter=U"])
            conflicting = conflict_result.stdout.strip().splitlines()
            git.abort_merge()
            _output({
                "success": False,
                "action_needed": "RESOLVE_CONFLICTS",
                "conflicting_files": conflicting,
            })
            return
        _error("Merge failed", result.stderr)
        return

    # Push to Apps Script and run tests on target branch
    push_result = clasp.push()
    if not push_result.success:
        git.reset_hard("HEAD~1")
        git.checkout(args.from_branch)
        _error("Push failed after merge", push_result.stderr)
        return

    import time
    time.sleep(config.log_wait_sec)

    run_result = clasp.run(config.test_runner_function)
    if run_result.success:
        test_report = parse_test_output(run_result.stdout)
        if test_report.success:
            git.push(args.to)
            _output({
                "success": True,
                "from_branch": args.from_branch,
                "to_branch": args.to,
                "merge_commit": git.head_hash(),
                "tests_passed": True,
                "pushed_to_github": True,
            })
            return
        else:
            # Tests failed on target — rollback merge
            git.reset_hard("HEAD~1")
            git.checkout(args.from_branch)
            _output({
                "success": False,
                "action_needed": "FIX_MERGE_TESTS",
                "test_report": json.loads(test_report.to_json()),
            })
            return

    # Run itself failed
    git.reset_hard("HEAD~1")
    git.checkout(args.from_branch)
    _error("Test execution failed after merge", run_result.stderr)


def cmd_release(args: argparse.Namespace, config: ProjectConfig) -> None:
    """Create a production release."""
    git = GitManager(config)
    clasp = ClaspWrapper(config)

    import shutil

    version = args.version
    summary = args.summary if hasattr(args, "summary") else f"Release v{version}"
    tag_name = f"v{version}"

    # Pre-flight checks
    current = git.current_branch()
    if current != config.develop_branch:
        _error(f"Must be on '{config.develop_branch}' to start a release, currently on '{current}'")

    if not git.is_clean():
        _error("Working tree is not clean")

    # Check tag doesn't exist
    existing_tag = git._git(["tag", "-l", tag_name])
    if existing_tag.stdout.strip():
        _error(f"Tag '{tag_name}' already exists")

    # Merge develop into main
    merge_msg = f"release(main): v{version} - {summary}"
    merge_result = git.merge(config.develop_branch, config.main_branch, merge_msg)
    if not merge_result.success:
        _error("Merge to main failed", merge_result.stderr)

    # Finalize changelog
    changelog_path = config.project_root / "CHANGELOG.md"
    finalize_release(changelog_path, version)
    git.add_all()
    git._git(["commit", "-m", f"docs(changelog): finalize v{version}"])

    # Tag
    git.tag(tag_name, f"Release {tag_name}: {summary}")

    # Push to production Apps Script
    push_result = clasp.push()
    if not push_result.success:
        # Rollback
        git._git(["tag", "-d", tag_name])
        git.reset_hard("HEAD~2")
        shutil.copy2(config.project_root / config.dev_config, config.project_root / ".clasp.json")
        git.checkout(config.develop_branch)
        _error("Push to production failed", push_result.stderr)

    # Create versioned deployment
    deploy_result = clasp.deploy(description=tag_name)

    # Smoke test
    import time
    time.sleep(config.log_wait_sec)

    run_result = clasp.run(config.test_runner_function)
    if run_result.success:
        test_report = parse_test_output(run_result.stdout)
        if test_report.success:
            # All good — push everything
            git.push(config.main_branch)
            git.push_tag(tag_name)

            # Create GitHub release
            notes = get_release_notes(changelog_path, version)
            notes_file = config.project_root / ".release_notes_tmp.md"
            notes_file.write_text(notes)
            gh_result = git.gh_create_release(tag_name, tag_name, str(notes_file))
            notes_file.unlink(missing_ok=True)

            _output({
                "success": True,
                "version": version,
                "tag": tag_name,
                "deployment_id": deploy_result.stdout if deploy_result.success else "unknown",
                "github_release_url": gh_result.stdout.strip() if gh_result.success else "",
                "smoke_tests_passed": True,
            })
            return

    # Smoke tests failed — rollback
    git._git(["tag", "-d", tag_name])
    git.reset_hard("HEAD~2")
    shutil.copy2(config.project_root / config.dev_config, config.project_root / ".clasp.json")
    git.checkout(config.develop_branch)
    _output({
        "success": False,
        "action_needed": "FIX_PRODUCTION_SMOKE",
        "rollback_performed": True,
        "error_detail": run_result.stderr if not run_result.success else "Smoke tests failed",
    })

"""runner.py — Feedback loop engine for push-run-fix cycles."""
from __future__ import annotations

import json
import shutil
import time
from pathlib import Path

from .config import ProjectConfig
from .git_manager import GitManager
from .clasp_wrapper import ClaspWrapper
from .log_parser import classify_error, parse_test_output
from .manifest import validate as validate_manifest
from .models import (
    CommandResult,
    ErrorCategory,
    IterationRecord,
    LoopReport,
    TestReport,
)
from .exceptions import PrePushValidationError


class FeedbackLoopRunner:
    """
    Orchestrates the push → run → evaluate → report cycle.

    Design principle: the runner does NOT attempt to fix code.
    It pushes, runs tests, and returns structured results.
    Claude Code is the intelligent agent that reads the results,
    decides on a fix, edits files, and invokes the runner again.
    """

    def __init__(self, config: ProjectConfig):
        self.config = config
        self.git = GitManager(config)
        self.clasp = ClaspWrapper(config)
        self.project_root = config.project_root

    def run(
        self,
        max_retries: int | None = None,
        delay: int | None = None,
    ) -> LoopReport:
        """
        Execute one iteration of the feedback loop.

        Despite the name 'run' and the max_retries parameter,
        this method performs a SINGLE push-run-evaluate cycle
        and returns the result. The max_retries parameter is
        tracked for reporting purposes so Claude Code knows
        how many attempts remain.

        The retry loop is driven by Claude Code externally:
        1. Claude Code calls gas-skill test-loop
        2. Runner pushes, runs tests, returns result
        3. If failure, Claude Code fixes code and calls again
        4. Repeat until success or max_retries exceeded

        Returns:
            LoopReport with success status and detailed results.
        """
        max_retries = max_retries or self.config.max_retries
        delay = delay or self.config.retry_delay_sec
        log_wait = self.config.log_wait_sec

        iterations: list[IterationRecord] = []
        record = IterationRecord(iteration=1)

        # ── Step 1: Auto-commit if dirty ─────────────────────
        if not self.git.is_clean():
            self.git.add_all()
            commit_result = self.git._git(
                ["commit", "-m", "fix(auto): pre-push auto-commit"]
            )
            if commit_result.success:
                record.committed = True
                record.commit_hash = self.git.head_hash()

        # ── Step 2: Pre-push validation ──────────────────────
        try:
            checks = self._validate_pre_push()
            failed_checks = [c for c in checks if not c["passed"]]
            if failed_checks:
                record.error_category = ErrorCategory.MANIFEST
                iterations.append(record)
                return LoopReport(
                    success=False,
                    total_iterations=1,
                    iterations=iterations,
                    branch=self.git.current_branch(),
                    final_commit=self.git.head_hash(),
                )
        except PrePushValidationError as e:
            record.error_category = ErrorCategory.MANIFEST
            iterations.append(record)
            return LoopReport(
                success=False,
                total_iterations=1,
                iterations=iterations,
                branch=self.git.current_branch(),
                final_commit=self.git.head_hash(),
            )

        # ── Step 3: Push to Apps Script ──────────────────────
        push_result = self.clasp.push()
        record.push_result = push_result

        if not push_result.success:
            record.error_category = classify_error(push_result.stderr)
            iterations.append(record)
            return LoopReport(
                success=False,
                total_iterations=1,
                iterations=iterations,
                branch=self.git.current_branch(),
                final_commit=self.git.head_hash(),
            )

        # ── Step 4: Wait for propagation ─────────────────────
        time.sleep(log_wait)

        # ── Step 5: Run tests ────────────────────────────────
        run_result = self.clasp.run(self.config.test_runner_function)

        if not run_result.success:
            # Execution itself failed (not a test failure)
            record.error_category = classify_error(run_result.stderr)
            iterations.append(record)
            return LoopReport(
                success=False,
                total_iterations=1,
                iterations=iterations,
                branch=self.git.current_branch(),
                final_commit=self.git.head_hash(),
            )

        # ── Step 6: Parse test results ───────────────────────
        test_report = parse_test_output(run_result.stdout)
        record.test_report = test_report

        if test_report.success:
            # All tests passed!
            if not self.git.is_clean():
                self.git.add_all()
                self.git._git(
                    ["commit", "-m",
                     f"test(pass): all {test_report.total} tests pass"]
                )
            record.committed = True
            record.commit_hash = self.git.head_hash()
            self.git.push()

            iterations.append(record)
            return LoopReport(
                success=True,
                total_iterations=1,
                iterations=iterations,
                branch=self.git.current_branch(),
                final_commit=self.git.head_hash(),
            )
        else:
            # Tests failed
            record.error_category = ErrorCategory.UNKNOWN
            # Classify based on first failing test
            for detail in test_report.details:
                if detail.status.value in ("FAIL", "ERROR"):
                    record.error_category = classify_error(detail.message)
                    break

            iterations.append(record)
            return LoopReport(
                success=False,
                total_iterations=1,
                iterations=iterations,
                branch=self.git.current_branch(),
                final_commit=self.git.head_hash(),
            )

    def _validate_pre_push(self) -> list[dict]:
        """
        Run pre-push validation checks.

        Returns a list of check results, each with:
            {"check": str, "passed": bool, "detail": str}
        """
        checks: list[dict] = []

        # 1. Not on main
        branch = self.git.current_branch()
        is_main = branch == self.config.main_branch
        checks.append({
            "check": "not_on_main",
            "passed": not is_main,
            "detail": f"Current branch: {branch}",
        })

        # 2. clasp config exists
        clasp_json = self.project_root / ".clasp.json"
        checks.append({
            "check": "clasp_config_exists",
            "passed": clasp_json.exists(),
            "detail": str(clasp_json),
        })

        # 3. Script ID matches environment
        if clasp_json.exists():
            active_id = self._read_script_id(clasp_json)
            expected_config_name = (
                self.config.prod_config if is_main else self.config.dev_config
            )
            expected_config_path = self.project_root / expected_config_name
            if expected_config_path.exists():
                expected_id = self._read_script_id(expected_config_path)
                checks.append({
                    "check": "script_id_matches_env",
                    "passed": active_id == expected_id,
                    "detail": (
                        f"active={active_id[:12]}... "
                        f"expected={expected_id[:12]}..."
                    ),
                })
            else:
                checks.append({
                    "check": "script_id_matches_env",
                    "passed": False,
                    "detail": f"Expected config not found: {expected_config_path}",
                })

        # 4. Clean working tree
        clean = self.git.is_clean()
        checks.append({
            "check": "clean_working_tree",
            "passed": clean,
            "detail": "All changes committed" if clean else "Uncommitted changes",
        })

        # 5. Valid manifest
        manifest_path = self.project_root / "appsscript.json"
        mv = validate_manifest(manifest_path)
        checks.append({
            "check": "valid_manifest",
            "passed": mv.ok,
            "detail": mv.message,
        })
        if mv.warnings:
            for w in mv.warnings:
                checks.append({
                    "check": "manifest_warning",
                    "passed": True,  # Warnings don't block
                    "detail": w,
                })

        return checks

    @staticmethod
    def _read_script_id(config_path: Path) -> str:
        """Read the scriptId from a .clasp.json file."""
        with open(config_path) as f:
            data = json.load(f)
        return data.get("scriptId", "")

ERROR_PATTERNS: list[tuple[re.Pattern, ErrorCategory]] = [
    (re.compile(r"not logged in|401|token.*expired", re.I), ErrorCategory.AUTH),
    (re.compile(r"SyntaxError|Unexpected token", re.I), ErrorCategory.SYNTAX),
    (re.compile(r"ReferenceError|is not defined", re.I), ErrorCategory.REFERENCE),
    (re.compile(r"TypeError|is not a function|cannot read propert", re.I), ErrorCategory.TYPE),
    (re.compile(r"Invalid manifest|malformed", re.I), ErrorCategory.MANIFEST),
    (re.compile(r"quota|rate.?limit|RESOURCE_EXHAUSTED", re.I), ErrorCategory.QUOTA),
    (re.compile(r"exceeded maximum execution time|timeout|timed?\s*out", re.I), ErrorCategory.TIMEOUT),
    (re.compile(r"ECONNREFUSED|ETIMEDOUT|network|fetch.*fail", re.I), ErrorCategory.NETWORK),
    (re.compile(r"permission denied|403|insufficient.*scope", re.I), ErrorCategory.PERMISSION),
    (re.compile(r"script not found|404|not found", re.I), ErrorCategory.NOT_FOUND),
]


def classify_error(error_text: str) -> ErrorCategory:
    """Classify an error string into a known category."""
    for pattern, category in ERROR_PATTERNS:
        if pattern.search(error_text):
            return category
    return ErrorCategory.UNKNOWN


# ── Test output parsing ───────────────────────────────────────

def parse_test_output(clasp_stdout: str) -> TestReport:
    """
    Parse the JSON output from runAllTests into a TestReport.

    clasp run output format:
        > runAllTests
        {"total": 5, "passed": 4, ...}

    Or for errors:
        > runAllTests
        Exception: ReferenceError: x is not defined
    """
    lines = clasp_stdout.strip().splitlines()

    # Try to find JSON in the output
    json_str = ""
    for i, line in enumerate(lines):
        stripped = line.strip()
        if stripped.startswith("{"):
            json_str = "\n".join(lines[i:])
            break

    if not json_str:
        # No JSON found — might be a runtime error
        error_text = clasp_stdout.strip()
        # Remove the "> functionName" prefix if present
        if error_text.startswith(">"):
            error_text = "\n".join(lines[1:]).strip()

        return TestReport(
            total=0,
            passed=0,
            failed=0,
            errors=1,
            success=False,
            details=[
                TestResult(
                    status=TestStatus.ERROR,
                    function="runAllTests",
                    message=f"No test output captured. Raw output: {error_text[:500]}",
                    timestamp="",
                    stack="",
                )
            ],
        )

    try:
        data = json.loads(json_str)
    except json.JSONDecodeError as e:
        return TestReport(
            total=0,
            passed=0,
            failed=0,
            errors=1,
            success=False,
            details=[
                TestResult(
                    status=TestStatus.ERROR,
                    function="runAllTests",
                    message=f"Invalid JSON in test output: {e}",
                    timestamp="",
                    stack="",
                )
            ],
        )

    details = [
        TestResult(
            status=TestStatus(d.get("status", "ERROR")),
            function=d.get("function", "unknown"),
            message=d.get("message", ""),
            timestamp=d.get("timestamp", ""),
            stack=d.get("stack", ""),
        )
        for d in data.get("details", [])
    ]

    return TestReport(
        total=data.get("total", 0),
        passed=data.get("passed", 0),
        failed=data.get("failed", 0),
        errors=data.get("errors", 0),
        success=data.get("success", False),
        details=details,
    )


# ── Log entry parsing ─────────────────────────────────────────

def parse_log_entries(clasp_logs_output: str) -> list[LogEntry]:
    """
    Parse the output of `clasp logs --json` into LogEntry objects.

    clasp logs --json returns entries in the format:
    [
      {
        "timestamp": "2026-03-22T14:30:00.000Z",
        "severity": "ERROR",
        "textPayload": "ReferenceError: x is not defined at ..."
      },
      ...
    ]

    Or in some versions, one JSON object per line.
    """
    entries: list[LogEntry] = []

    if not clasp_logs_output.strip():
        return entries

    # Try parsing as JSON array first
    try:
        data = json.loads(clasp_logs_output)
        if isinstance(data, list):
            for item in data:
                entries.append(_parse_single_log(item))
            return entries
    except json.JSONDecodeError:
        pass

    # Try parsing as newline-delimited JSON
    for line in clasp_logs_output.strip().splitlines():
        line = line.strip()
        if not line or line.startswith(">"):
            continue
        try:
            item = json.loads(line)
            entries.append(_parse_single_log(item))
        except json.JSONDecodeError:
            # Not JSON — try to parse as plain text log
            entry = _parse_plain_log_line(line)
            if entry:
                entries.append(entry)

    return entries


def _parse_single_log(item: dict) -> LogEntry:
    """Parse a single JSON log object into a LogEntry."""
    message = item.get("textPayload", item.get("message", ""))
    severity_str = item.get("severity", "INFO").upper()

    try:
        severity = Severity(severity_str)
    except ValueError:
        severity = Severity.INFO

    # Try to extract function name and stack trace from message
    function_name = ""
    stack_trace = ""

    # Pattern: "at functionName (File:line:col)"
    stack_match = re.search(r"at (\w+)\s*\(", message)
    if stack_match:
        function_name = stack_match.group(1)

    # If message contains newlines, the part after the first line is stack
    if "\n" in message:
        parts = message.split("\n", 1)
        message = parts[0]
        stack_trace = parts[1].strip()

    return LogEntry(
        timestamp=item.get("timestamp", ""),
        severity=severity,
        message=message,
        function_name=function_name,
        stack_trace=stack_trace,
    )


def _parse_plain_log_line(line: str) -> LogEntry | None:
    """
    Attempt to parse a plain text log line.

    Expected format: "TIMESTAMP SEVERITY MESSAGE"
    Example: "2026-03-22 14:30:00 ERROR ReferenceError: x is not defined"
    """
    # Pattern: date time severity message
    pattern = re.compile(
        r"(\d{4}-\d{2}-\d{2}[\sT]\d{2}:\d{2}:\d{2}[.\d]*Z?)\s+"
        r"(DEBUG|INFO|WARNING|ERROR|CRITICAL)\s+"
        r"(.+)",
        re.I,
    )
    match = pattern.match(line)
    if match:
        try:
            severity = Severity(match.group(2).upper())
        except ValueError:
            severity = Severity.INFO

        return LogEntry(
            timestamp=match.group(1),
            severity=severity,
            message=match.group(3),
        )

    return None

# Google Apps Script Integration Skill

## Overview

You have access to a Python CLI tool called `gas-skill` that manages
the full lifecycle of Google Apps Script development. All GAS operations
MUST go through this tool — never call `clasp` or `git` directly.

## Quick Reference

```bash
# Check prerequisites
gas-skill preflight

# Project status
gas-skill status

# Create a feature branch
gas-skill branch create <name>

# Commit changes
gas-skill commit -m "<message>" --type <feat|fix|test|...> [--scope <scope>]

# Push to Apps Script and run tests
gas-skill test-loop

# Merge when tests pass
gas-skill merge --from <branch> --to develop

# Release to production
gas-skill release --version <X.Y.Z> --summary "<text>"

#+END_SRC

* 11. Security Considerations

** 11.1 Credential Management

*** OAuth Tokens

The =~/.clasprc.json= file contains OAuth tokens for Google API access.
This file must never be committed to Git. The =.gitignore= template
excludes it by default. The token has the scopes defined in the
=appsscript.json= manifest and can access any Apps Script project owned
by the authenticated user.

*** GitHub Tokens

The GitHub CLI (=gh=) stores its authentication token in the system
keychain or in =~/.config/gh/hosts.yml=. This is managed by =gh auth
login= and should never be committed to Git.

*** Credential Rotation

OAuth tokens should be rotated periodically by running =clasp logout=
followed by =clasp login=. The tool does not manage token rotation
automatically but will detect expired tokens and report them as
=AUTH= category errors.

** 11.2 Script ID Protection

The =.clasp.json= file is excluded from Git because it contains the
active script ID and could cause environment confusion if committed.
The environment-specific configs (=.clasp.prod.json= and =.clasp.dev.json=)
are committed to Git because they are part of the project configuration,
but they only contain script IDs (not credentials).

** 11.3 Branch Protection Enforcement

The Python orchestration layer enforces branch protection rules in software.
These rules prevent direct commits to =main=, prevent force pushes to
protected branches, and ensure the correct =.clasp.json= is active for
each branch. For additional protection, GitHub branch protection rules
should be configured on the repository as described in Section 4.6.3.

** 11.4 Production Deployment Safety

Production deployments only occur through the =gas-skill release= command,
which enforces the following safety chain: all tests must pass on =develop=,
the merge to =main= must succeed, the push to the production Apps Script
must succeed, smoke tests on production must pass, and automatic rollback
occurs if any step fails.

* 12. Failure Modes and Recovery

** 12.1 Failure Mode Table

| Failure Mode                    | Detection                              | Recovery Action                                        |
|---------------------------------+----------------------------------------+--------------------------------------------------------|
| clasp auth expired              | AUTH error category                    | User runs =clasp login= manually                      |
| GitHub auth expired             | =gh auth status= fails                | User runs =gh auth login= manually                    |
| Push fails (syntax error)       | SYNTAX error category                  | Claude Code fixes syntax, re-runs test-loop            |
| Push fails (manifest error)     | MANIFEST error category                | Claude Code fixes appsscript.json                      |
| Tests fail                      | TestReport.success == false            | Claude Code reads details, fixes code, re-runs         |
| Runtime error during test run   | clasp run exit code != 0               | Claude Code reads error, fixes code                    |
| Merge conflict                  | CONFLICT in merge output               | Claude Code resolves conflicts, re-runs merge          |
| Production smoke test fail      | TestReport on main fails               | Auto-rollback, fix on develop, re-release              |
| Network error                   | NETWORK error category                 | Retry after delay                                      |
| Quota exceeded                  | QUOTA error category                   | Wait and retry, or optimize code                       |
| Execution timeout               | TIMEOUT error category                 | Optimize code or break into batches                    |
| Max retries exceeded            | Loop counter >= max_retries            | Report to user, suggest manual investigation           |
| .clasp.json missing             | Pre-push validation fails              | Run =gas-skill branch switch= to regenerate            |
| Wrong environment active        | Script ID mismatch in validation       | Run =gas-skill branch switch= to correct               |
| Git remote unreachable          | git push fails                         | Check network, retry                                   |
| Corrupted working tree          | git status shows unexpected state      | =git stash=, switch branch, investigate                |

** 12.2 Automatic Rollback Protocol

The release command implements automatic rollback if production
deployment fails. The rollback sequence is:

#+BEGIN_SRC
1. Delete the local tag:       git tag -d v{version}
2. Reset main to pre-merge:    git reset --hard HEAD~2
3. Restore dev clasp config:   cp .clasp.dev.json .clasp.json
4. Switch to develop:          git checkout develop
5. Report failure with:        rollback_performed=True
#+END_SRC

This ensures that =main= is never left in a broken state. The production
Apps Script project may have received the broken push, but since no
versioned deployment was finalized, the previous deployment remains active.

** 12.3 Manual Recovery Commands

For situations that the automated tool cannot handle, the following
manual recovery steps are available:

#+BEGIN_SRC bash
# Reset to last known good state on current branch
git reset --hard HEAD~1

# Abort a failed merge
git merge --abort

# Force-sync with remote (caution: loses local changes)
git fetch origin
git reset --hard origin/{branch}

# Re-authenticate clasp
clasp logout
clasp login --creds /path/to/credentials.json

# Re-authenticate GitHub CLI
gh auth logout
gh auth login

# Pull latest from Apps Script (overwrite local)
clasp pull

# Check what clasp thinks the project is
clasp status
#+END_SRC

#+BEGIN_SRC python
"""models.py — Data models for gas-skill."""
from __future__ import annotations

import json
from dataclasses import dataclass, field, asdict
from enum import Enum
from typing import Any


# ── Enums ──────────────────────────────────────────────────────

class ErrorCategory(str, Enum):
    """Classification of errors from clasp operations."""
    AUTH = "AUTH"
    REFERENCE = "REFERENCE"
    TYPE = "TYPE"
    SYNTAX = "SYNTAX"
    MANIFEST = "MANIFEST"
    QUOTA = "QUOTA"
    TIMEOUT = "TIMEOUT"
    NETWORK = "NETWORK"
    PERMISSION = "PERMISSION"
    NOT_FOUND = "NOT_FOUND"
    UNKNOWN = "UNKNOWN"


class TestStatus(str, Enum):
    """Status of an individual test."""
    PASS = "PASS"
    FAIL = "FAIL"
    ERROR = "ERROR"


class Severity(str, Enum):
    """Log entry severity levels."""
    DEBUG = "DEBUG"
    INFO = "INFO"
    WARNING = "WARNING"
    ERROR = "ERROR"
    CRITICAL = "CRITICAL"


# ── Command Result ─────────────────────────────────────────────

@dataclass
class CommandResult:
    """Result of executing an external command."""
    command: str
    exit_code: int
    stdout: str
    stderr: str
    success: bool
    duration_sec: float

    def to_dict(self) -> dict:
        return asdict(self)

    def to_json(self) -> str:
        return json.dumps(self.to_dict(), indent=2)


# ── Test Models ────────────────────────────────────────────────

@dataclass
class TestResult:
    """Result of a single test function."""
    status: TestStatus
    function: str
    message: str
    timestamp: str = ""
    stack: str = ""

    def to_dict(self) -> dict:
        d = asdict(self)
        d["status"] = self.status.value
        return d


@dataclass
class TestReport:
    """Aggregated test results from runAllTests."""
    total: int
    passed: int
    failed: int
    errors: int
    success: bool
    details: list[TestResult] = field(default_factory=list)

    def to_dict(self) -> dict:
        return {
            "total": self.total,
            "passed": self.passed,
            "failed": self.failed,
            "errors": self.errors,
            "success": self.success,
            "details": [d.to_dict() for d in self.details],
        }

    def to_json(self) -> str:
        return json.dumps(self.to_dict(), indent=2)

    @property
    def failing_tests(self) -> list[TestResult]:
        """Return only failed and errored tests."""
        return [
            d for d in self.details
            if d.status in (TestStatus.FAIL, TestStatus.ERROR)
        ]

    @property
    def summary(self) -> str:
        """Human-readable summary."""
        return (
            f"{self.passed}/{self.total} passed, "
            f"{self.failed} failed, "
            f"{self.errors} errors"
        )


# ── Log Models ─────────────────────────────────────────────────

@dataclass
class LogEntry:
    """A single log entry from Apps Script execution."""
    timestamp: str = ""
    severity: Severity = Severity.INFO
    message: str = ""
    function_name: str = ""
    stack_trace: str = ""

    def to_dict(self) -> dict:
        d = asdict(self)
        d["severity"] = self.severity.value
        return d


# ── Iteration Models ──────────────────────────────────────────

@dataclass
class IterationRecord:
    """Record of a single push-run-evaluate cycle."""
    iteration: int = 0
    committed: bool = False
    commit_hash: str = ""
    push_result: CommandResult | None = None
    test_report: TestReport | None = None
    error_category: ErrorCategory = ErrorCategory.UNKNOWN

    def to_dict(self) -> dict:
        return {
            "iteration": self.iteration,
            "committed": self.committed,
            "commit_hash": self.commit_hash,
            "push_result": self.push_result.to_dict() if self.push_result else None,
            "test_report": self.test_report.to_dict() if self.test_report else None,
            "error_category": self.error_category.value,
        }


@dataclass
class LoopReport:
    """Complete report from a feedback loop execution."""
    success: bool
    total_iterations: int
    iterations: list[IterationRecord] = field(default_factory=list)
    branch: str = ""
    final_commit: str = ""
    action_needed: str = ""
    error_detail: str = ""

    def to_dict(self) -> dict:
        return {
            "success": self.success,
            "total_iterations": self.total_iterations,
            "iterations": [i.to_dict() for i in self.iterations],
            "branch": self.branch,
            "final_commit": self.final_commit,
            "action_needed": self.action_needed,
            "error_detail": self.error_detail,
        }

    def to_json(self) -> str:
        return json.dumps(self.to_dict(), indent=2, default=str)


# ── Project Status Model ──────────────────────────────────────

@dataclass
class ProjectStatus:
    """Comprehensive project status."""
    branch: str = ""
    environment: str = "dev"
    script_id: str = ""
    clean_tree: bool = True
    uncommitted_files: list[str] = field(default_factory=list)
    latest_commit_hash: str = ""
    latest_commit_message: str = ""
    latest_tag: str = ""
    ahead: int = 0
    behind: int = 0
    local_branches: list[str] = field(default_factory=list)
    remote_branches: list[str] = field(default_factory=list)
    clasp_config_valid: bool = False
    clasp_config_env_match: bool = False

    def to_dict(self) -> dict:
        return asdict(self)

    def to_json(self) -> str:
        return json.dumps(self.to_dict(), indent=2)

* 14. Development Workflow

This section defines the standard development workflow for all work on
a =gas-skill= managed project. These procedures apply equally to human
developers working through Claude Code and to Claude Code operating
autonomously. Every code change, no matter how small, follows this
workflow to ensure traceability, environment safety, and test coverage.

** 14.1 Starting New Work

Every unit of work begins with a feature branch. No code is ever written
directly on =develop= or =main=.

#+BEGIN_SRC bash
gas-skill branch create <descriptive-name>
#+END_SRC

This command performs the following sequence atomically: it checks out
=develop= and pulls the latest changes from =origin=, creates a new
branch named =feature/<descriptive-name>= from the tip of =develop=,
copies =.clasp.dev.json= to =.clasp.json= so that all subsequent pushes
target the development Apps Script project, and pushes the new branch
to =origin= with tracking enabled.

The branch name should be concise but descriptive. Use lowercase words
separated by hyphens. Examples include =column-average=,
=email-notification-trigger=, and =sheet-validation-refactor=.

For other branch types, use the corresponding subcommand or prefix:

#+BEGIN_SRC bash
# Bugfix on develop
gas-skill branch create <name>          # uses feature/ prefix by default

# Hotfix on main (critical production fix)
gas-skill hotfix create <name>          # creates hotfix/<name> from main
#+END_SRC

** 14.2 Writing Code

All Apps Script source files live in the =src/= directory and use the
=.js= extension. The Apps Script runtime uses the V8 engine, so modern
JavaScript syntax (=const=, =let=, arrow functions, template literals,
destructuring) is fully supported. However, ES modules (=import=/=export=)
are not available; all functions share a single global namespace.

*** 14.2.1 Source File Conventions

Each source file should have a clear responsibility. Use a JSDoc header
comment at the top of each file to describe its purpose. All public
functions should have JSDoc annotations documenting parameters, return
types, and behavior.

#+BEGIN_SRC javascript
/**
 * SheetUtils.js — Utility functions for spreadsheet operations.
 */

/**
 * Calculate the average of all numeric values in a column.
 * @param {GoogleAppsScript.Spreadsheet.Sheet} sheet - The target sheet.
 * @param {number} column - The 1-based column index.
 * @returns {number} The arithmetic mean, or 0 if the column is empty.
 */
function columnAverage(sheet, column) {
  var data = sheet.getRange(1, column, sheet.getLastRow(), 1).getValues();
  var nums = data
    .map(function(row) { return row[0]; })
    .filter(function(v) { return typeof v === "number" && !isNaN(v); });

  if (nums.length === 0) return 0;

  var sum = nums.reduce(function(a, b) { return a + b; }, 0);
  return sum / nums.length;
}
#+END_SRC

*** 14.2.2 Writing Tests

Every new function or behavior change must have corresponding tests in
=src/Tests.js=. Tests use the lightweight test harness defined in
Section 8.2 and follow these conventions:

All test function names begin with =test_= so the harness can discover
them automatically. Each test function returns an object with exactly
two properties: =passed= (a boolean) and =message= (a string describing
the outcome). Test functions use the provided assertion helpers rather
than manual comparisons.

The available assertion helpers are:

=assertEqual(actual, expected, label)= checks strict equality with the
===== operator. =assertTrue(value, label)= checks that a value is truthy.
=assertFalse(value, label)= checks that a value is falsy.
=assertThrows(fn, expectedMessage, label)= checks that a function throws
an error, optionally matching a substring in the error message.
=assertDeepEqual(actual, expected, label)= checks structural equality by
comparing JSON serializations.

#+BEGIN_SRC javascript
function test_column_average_normal() {
  // Simulate a column of numbers
  var values = [10, 20, 30, 40, 50];
  var sum = values.reduce(function(a, b) { return a + b; }, 0);
  var avg = sum / values.length;
  return assertEqual(avg, 30, "average of 10-50");
}

function test_column_average_empty() {
  var nums = [];
  var result = nums.length === 0 ? 0 : nums.reduce(function(a, b) { return a + b; }, 0) / nums.length;
  return assertEqual(result, 0, "empty column returns 0");
}

function test_column_average_mixed_types() {
  var data = [10, "hello", 20, null, 30];
  var nums = data.filter(function(v) { return typeof v === "number" && !isNaN(v); });
  var avg = nums.reduce(function(a, b) { return a + b; }, 0) / nums.length;
  return assertEqual(avg, 20, "mixed types filters non-numbers");
}
#+END_SRC

*** 14.2.3 Test Design Guidelines

Tests should cover the happy path (normal expected inputs), edge cases
(empty inputs, single elements, boundary values), error conditions
(invalid inputs that should be handled gracefully), and type coercion
scenarios (since Apps Script receives data from spreadsheets which may
contain mixed types).

Because tests run in the remote Apps Script environment, they have access
to all Google services. However, tests that modify live data (creating
sheets, sending emails, writing to documents) should clean up after
themselves or use dedicated test fixtures. Tests should be idempotent:
running them multiple times should produce the same results.

** 14.3 Committing Changes

After writing or modifying code, create a conventional commit:

#+BEGIN_SRC bash
gas-skill commit -m "description of change" --type feat --scope module
#+END_SRC

The =--type= flag determines the conventional commit prefix and the
CHANGELOG section where the entry will appear. The available types and
their meanings are:

=feat= is for new features or capabilities and maps to the "Added"
section. =fix= is for bug fixes and maps to "Fixed". =refactor= is for
code restructuring without behavior change and maps to "Changed".
=test= is for adding or updating tests and maps to "Changed". =docs=
is for documentation changes and maps to "Changed". =perf= is for
performance improvements and maps to "Changed". =style= is for
formatting or code style changes and maps to "Changed". =chore= is
for build system or tooling changes and maps to "Changed".

The =--scope= flag is optional and adds context to the commit message.
It should name the module, file, or area of the codebase being changed.

The commit command performs the following: stages all changes with
=git add -A=, creates a commit with the conventional message format
(e.g., =feat(sheets): add column average calculator=), pushes the commit
to the remote branch, adds an entry to the =[Unreleased]= section of
=CHANGELOG.md= under the appropriate subsection, and commits the
changelog update as a separate commit.

** 14.4 Testing

After committing, run the feedback loop to push code to the Apps Script
environment and execute tests:

#+BEGIN_SRC bash
gas-skill test-loop
#+END_SRC

The test-loop performs a single push-run-evaluate cycle and returns
structured results. The steps are executed in order:

First, the runner auto-commits any uncommitted changes with the message
=fix(auto): pre-push auto-commit=. This ensures the Git history always
reflects what was pushed to Apps Script. Second, the runner validates
the project state by checking the manifest (=appsscript.json=), the
environment configuration (correct =.clasp.json= for the current branch),
and the working tree cleanliness. Third, the runner executes =clasp push=
to upload the source files to the development Apps Script project.
Fourth, the runner waits a configurable number of seconds (default 3)
for the pushed code to propagate in Google's infrastructure. Fifth,
the runner executes =clasp run runAllTests= to invoke the test harness
remotely. Sixth, the runner parses the JSON output from =runAllTests=
into a structured =TestReport= and returns it.

The output is always JSON and contains a =success= boolean at the top
level, a =test_report= object with individual test results, and an
=error_category= classification if something failed.

** 14.5 Handling Test Failures

When =test-loop= returns a failure, the response JSON contains all the
information needed to diagnose and fix the problem. The handling
procedure is:

Read the JSON output carefully. The =error_category= field classifies
the type of error: =SYNTAX= means the code has a syntax error that
prevented execution, =REFERENCE= means a variable or function name is
misspelled or undefined, =TYPE= means a method was called on the wrong
type of value, =MANIFEST= means the =appsscript.json= file is invalid,
=AUTH= means the authentication token has expired, =QUOTA= means a
Google API quota was exceeded, =TIMEOUT= means the script exceeded the
maximum execution time, and =NETWORK= means a network request failed.

The =test_report.details= array contains one entry per test function.
Each entry has a =status= (=PASS=, =FAIL=, or =ERROR=), a =message=
describing what went wrong, and a =stack= trace if a runtime error
occurred. Focus on the first failing test and fix the underlying issue.

After fixing the code, run =gas-skill test-loop= again. This cycle of
fix-and-test continues until all tests pass or the maximum retry count
(default 5) is exceeded. If maximum retries are exceeded, step back and
reconsider the approach rather than making incremental fixes.

#+BEGIN_SRC
Iteration 1: test-loop → FAIL (test_empty_column: Expected 0, got NaN)
             Fix: add empty array guard in columnAverage()
Iteration 2: test-loop → FAIL (test_mixed_types: Expected 20, got 12.5)
             Fix: filter non-numeric values before averaging
Iteration 3: test-loop → PASS (5/5 tests pass)
#+END_SRC

** 14.6 Merging

After all tests pass on the feature branch, merge into =develop=:

#+BEGIN_SRC bash
gas-skill merge --from feature/<name> --to develop
#+END_SRC

The merge command performs the following sequence: checks out the target
branch (=develop=) and pulls the latest changes, performs a no-fast-forward
merge (=--no-ff=) to preserve branch history, swaps =.clasp.json= to the
development configuration (since =develop= uses the dev project), pushes
the merged code to the dev Apps Script project via =clasp push=, runs
the full test suite on the target branch to verify the merge didn't
break anything, and only pushes to GitHub if all tests pass on the
target branch.

If the merge produces conflicts, the command aborts the merge and
returns a JSON response with =action_needed: "RESOLVE_CONFLICTS"= and
a list of conflicting files. Resolve the conflicts in the source files,
then re-run the merge command.

If tests fail after the merge, the command rolls back the merge commit
with =git reset --hard HEAD~1= and returns to the feature branch. This
ensures =develop= is never left in a broken state.

** 14.7 Releasing

When =develop= has accumulated enough changes for a release, create a
production release from the =develop= branch:

#+BEGIN_SRC bash
gas-skill release --version X.Y.Z --summary "Release description"
#+END_SRC

The version number follows Semantic Versioning: increment the major
version for breaking changes, the minor version for new features, and
the patch version for bug fixes.

The release command performs the following sequence: verifies the current
branch is =develop= and the working tree is clean, merges =develop= into
=main= with a no-fast-forward merge, moves all entries from the
=[Unreleased]= section of =CHANGELOG.md= under a new version heading,
creates an annotated Git tag (=v{X.Y.Z}=), swaps =.clasp.json= to the
production configuration, pushes the code to the production Apps Script
project via =clasp push=, creates a versioned deployment in Apps Script,
runs the full test suite on the production project as a smoke test,
pushes the tag and =main= branch to GitHub, and creates a GitHub release
with the changelog entries as release notes.

If the smoke test fails on production, the command performs an automatic
rollback: it deletes the local tag, resets =main= to its pre-merge state,
restores the development =.clasp.json=, and switches back to =develop=.
The production Apps Script project may have received the broken push,
but since the versioned deployment was not finalized, the previous
deployment remains active for any users or triggers relying on it.

** 14.8 Critical Rules

These rules are absolute and must never be violated, regardless of
time pressure or convenience:

Never call =clasp= or =git= directly. Always use =gas-skill=. The tool
maintains invariants (environment switching, branch protection, commit
formatting) that direct CLI calls would bypass.

Never commit or push to =main= directly. The =main= branch represents
production. All changes reach =main= through the release or hotfix
process, which includes testing and rollback safety.

Never push to the production Apps Script project from a non-main branch.
The production project should only receive code that has been tested on
=develop= and merged through the release process.

Always create a feature branch for new work. Even single-line changes
get a feature branch. This ensures every change is testable, reviewable,
and revertible.

Always write tests for new functionality. Untested code is assumed to
be broken. The test-loop cannot protect against regressions if there
are no tests to catch them.

Always run =gas-skill test-loop= before merging. The merge command
runs tests on the target branch, but running tests on the feature
branch first catches problems earlier and keeps the fix history on
the feature branch where it belongs.

Always check =gas-skill status= if unsure about the current state.
The status command shows the active branch, environment, script ID,
working tree cleanliness, and remote sync status. When in doubt, check
status first.

** 14.9 Error Recovery

When something goes wrong, the appropriate recovery action depends on
the error category:

For authentication errors (=AUTH= category), the user must run
=clasp login= manually. The gas-skill tool cannot perform interactive
OAuth flows. After re-authenticating, retry the failed operation.

For merge conflicts, read the structured conflict details returned by
the merge command. Resolve the conflicts in the source files by editing
them directly, then run the merge command again. The tool will detect
the resolved state and proceed.

For production smoke test failures, the tool automatically rolls back
=main= to its pre-release state. The fix should be applied on =develop=,
tested thoroughly, and a new release attempted. Do not try to fix
production directly.

For maximum retries exceeded, review the iteration history in the loop
report. If the same error recurs across iterations, the fix approach
is likely wrong. Step back, reconsider the implementation strategy,
and try a fundamentally different approach rather than incremental
adjustments.

For network errors (=NETWORK= category), wait a few seconds and retry.
These are typically transient. If they persist, check network
connectivity and Google's service status.

For quota errors (=QUOTA= category), wait for the quota to reset (usually
per-minute or per-day quotas) or optimize the code to make fewer API
calls.

** 14.10 Output Format Contract

All =gas-skill= commands return JSON to stdout. This is a design
contract — the tool never returns unstructured text. Errors are reported
as JSON with =success: false=, not as stderr output or non-zero exit
codes alone.

The key fields present in every response are:

=success= is a boolean indicating whether the operation completed
successfully. This is the primary field to check.

=action_needed= is a string that appears only on failure. It describes
what should be done next in machine-readable terms, such as
=FIX_TEST_FAILURE=, =RESOLVE_CONFLICTS=, =FIX_MERGE_TESTS=, or
=FIX_PRODUCTION_SMOKE=.

=error_category= is a string from the =ErrorCategory= enum that
classifies the type of error. This helps determine the appropriate
recovery action.

=error_detail= is a string containing the raw error text from the
underlying tool (clasp, git, or gh). This provides the specific
information needed to diagnose the problem.

Additional fields vary by command and are documented in the CLI
specification (Section 9.8) and the data models (Appendix D).

Google Apps Script projects through Claude Code. It covers the full
lifecycle from project initialization through production release, with
automated testing, structured error handling, Git-based version control,
and automatic environment switching.

The design prioritizes:
- **Safety**: Branch protection, environment isolation, automatic rollback
- **Traceability**: Every action produces a Git commit and structured JSON output
- **Claude Code agency**: The tool reports; Claude Code decides and acts
- **Simplicity**: Single CLI tool wrapping clasp + git + gh
- **Reliability**: Classified errors, pre-flight checks, validation at every step
